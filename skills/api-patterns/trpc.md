# Princípios de tRPC

> Segurança de tipos ponta a ponta para monorepos em TypeScript.

## Quando Usar

```
✅ Opção perfeita:
├── TypeScript em ambas as pontas
├── Estrutura de monorepo
├── Ferramentas internas
├── Desenvolvimento rápido
├── Segurança de tipos é crítica

❌ Opção ruim:
├── Clientes que não usam TypeScript
├── API pública
├── Necessidade de convenções REST
├── Backends em múltiplas linguagens
```

## Benefícios Chave

```
Por que tRPC:
├── Zero manutenção de schema
├── Inferência de tipos ponta a ponta
├── Autocomplete da IDE em toda a stack
├── Mudanças na API refletidas instantaneamente
└── Sem etapa de geração de código (code-gen)
```

## Padrões de Integração

```
Configurações comuns:
├── Next.js + tRPC (mais comum)
├── Monorepo com tipos compartilhados
├── Remix + tRPC
└── Qualquer frontend + backend em TS
```
