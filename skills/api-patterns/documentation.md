# Princípios de Documentação de API

> Boas docs = desenvolvedores felizes = adoção da API.

## Essenciais do OpenAPI/Swagger

```
Inclua:
├── Todos os endpoints com exemplos
├── Schemas de requisição/resposta
├── Requisitos de autenticação
├── Formatos de resposta de erro
└── Informações de limite de taxa (Rate limiting)
```

## Uma Boa Documentação Possui

```
Essenciais:
├── Início Rápido / Getting Started
├── Guia de autenticação
├── Referência completa da API
├── Guia de tratamento de erros
├── Exemplos de código (múltiplas linguagens)
└── Registro de mudanças (Changelog)
```
