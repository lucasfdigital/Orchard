# Princípios de Game Design

> Design thinking para jogos engajadores.

---

## 1. Design do Core Loop

### O Teste de 30 Segundos

```
Todo jogo precisa de um loop divertido de 30 segundos:
1. AÇÃO        → O jogador faz algo
2. FEEDBACK    → O jogo responde
3. RECOMPENSA  → O jogador se sente bem
4. REPETIR
```

### Exemplos de Loop

| Gênero | Core Loop |
| :--- | :--- |
| Plataforma | Correr → Pular → Pousar → Coletar |
| Shooter | Mirar → Atirar → Matar → Saquear (Loot) |
| Puzzle | Observar → Pensar → Resolver → Avançar |
| RPG | Explorar → Lutar → Subir de Nível → Equipar |

## 2. Documento de Design de Jogo (GDD)

### Seções Essenciais

| Seção | Conteúdo |
| :--- | :--- |
| **Pitch** | Descrição em uma única frase |
| **Core Loop** | Jogabilidade de 30 segundos |
| **Mecânicas** | Como os sistemas funcionam |
| **Progressão** | Como o jogador avança |
| **Estilo de Arte** | Direção visual |
| **Áudio** | Direção de som |

### Princípios

- Mantenha-o vivo (atualize regularmente)
- Visuais ajudam a comunicar
- Menos é mais (comece pequeno)

## 3. Psicologia do Jogador

### Tipos de Motivação

| Tipo | Motivado Por |
| :--- | :--- |
| **Conquistador (Achiever)** | Objetivos, conclusão |
| **Explorador** | Descoberta, segredos |
| **Socializador** | Interação, comunidade |
| **Assassino (Killer)** | Competição, dominância |

### Esquemas de Recompensa

| Esquema | Efeito | Uso |
| :--- | :--- | :--- |
| **Fixo** | Previsível | Recompensas de marcos (milestones) |
| **Variável** | Viciante | Loot drops |
| **Razão** | Baseado em esforço | Jogos de "grind" |

---

## 4. Balanceamento de Dificuldade

### Estado de Flow (Fluxo)

```
Muito Difícil → Frustração → Sair
Muito Fácil   → Tédio      → Sair
Na Medida     → Flow       → Engajamento
```

### Estratégias de Balanceamento

| Estratégia | Como |
| :--- | :--- |
| **Dinâmico** | Ajustar à habilidade do jogador |
| **Seleção** | Deixar o jogador escolher |
| **Acessibilidade** | Opções para todos |

---

## 5. Design de Progressão

### Tipos de Progressão

| Tipo | Exemplo |
| :--- | :--- |
| **Habilidade (Skill)** | O jogador fica melhor |
| **Poder** | O personagem fica mais forte |
| **Conteúdo** | Novas áreas são desbloqueadas |
| **História** | A narrativa avança |

### Princípios de Ritmo (Pacing)

- Vitórias iniciais (fisgue rápido)
- Aumentar gradualmente o desafio
- Momentos de descanso entre intensidades
- Escolhas significativas

---

## 6. Anti-Padrões

| ❌ Não Faça | ✅ Faça |
| :--- | :--- |
| Design isolado | Teste (playtest) constantemente |
| Polimento antes da diversão | Prototipe primeiro |
| Forçar uma única forma de jogar | Permita a expressão do jogador |
| Punir excessivamente | Recompense o progresso |

---

> **Lembre-se:** A diversão é descoberta através da iteração, não projetada no papel.
